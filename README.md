# Hermes VK Connector

Плагин-адаптер для [Hermes Agent](https://github.com/NousResearch/hermes-agent) (Nous Research), добавляющий поддержку ВКонтакте как канала общения с AI-агентом.

---

## Зачем это нужно

Hermes Agent — это автономный AI-агент с долгосрочной памятью, инструментами (поиск, код, файлы, браузер) и поддержкой нескольких мессенджеров. Из коробки он умеет работать с Telegram, Discord, Slack, WhatsApp и Signal.

Этот плагин добавляет **ВКонтакте** — чтобы пользователи могли общаться с агентом через личные сообщения или беседы ВК, не устанавливая отдельных приложений.

**Типичный сценарий:** вы настраиваете Hermes на своём сервере или домашнем компьютере, создаёте VK-сообщество, подключаете токен — и пишете боту прямо из ВК, как обычному собеседнику. Агент отвечает, помнит контекст между сессиями, умеет выполнять задачи (искать информацию, писать код, работать с файлами) и поддерживает голосовые сообщения.

---

## Как это работает

### Архитектура Hermes Gateway

Hermes Agent использует компонент **Gateway** — маршрутизатор сообщений между внешними платформами и ядром агента. Каждая платформа подключается через адаптер, наследующий базовый класс `BasePlatformAdapter`.

```
┌─────────────────────────────────────────────────────────┐
│                      Hermes Agent                       │
│                                                         │
│  ┌──────────┐    ┌─────────┐    ┌───────────────────┐  │
│  │  Agent   │◄──►│ Gateway │◄──►│  Platform Adapter │  │
│  │  Core    │    │ Router  │    │  (этот плагин)    │  │
│  └──────────┘    └─────────┘    └─────────┬─────────┘  │
└─────────────────────────────────────────┬─┘            │
                                          │               │
                              ┌───────────▼────────────┐  │
                              │     VK Long Poll API    │  │
                              └───────────┬────────────┘  │
                                          │
                              ┌───────────▼────────────┐
                              │   Пользователь в VK    │
                              └────────────────────────┘
```

Плагин регистрируется в системе через функцию `register(ctx)` и загружается автоматически при старте `hermes gateway`.

### Получение сообщений: Long Poll

VK не поддерживает WebSocket для ботов сообществ. Вместо этого используется **Long Poll** — механизм длинного HTTP-запроса: сервер держит соединение открытым до появления нового события (или до таймаута).

Поскольку `VkBotLongPoll.listen()` — это блокирующий генератор, он запускается в отдельном daemon-потоке. Новые события передаются в основной asyncio event loop через `asyncio.run_coroutine_threadsafe`:

```
  daemon thread                    asyncio event loop
  ─────────────                    ──────────────────
  VkBotLongPoll.listen()
       │ (блокирует до события)
       ▼
  получено событие
       │
       └──► run_coroutine_threadsafe(handle_message, loop)
                                          │
                                          ▼
                                    BasePlatformAdapter
                                    .handle_message()
                                          │
                                          ▼
                                    Hermes Agent Core
```

При обрыве соединения поток автоматически переподключается с экспоненциальным backoff (1s → 2s → 4s → ... → 60s).

### Отправка сообщений

Все исходящие операции (текст, фото, файлы, голосовые) используют `asyncio.to_thread()` — синхронные вызовы VK API выполняются в пуле потоков, не блокируя event loop агента.

```
Hermes Agent
    │
    ▼
adapter.send(chat_id, text)
    │
    └──► asyncio.to_thread(vk_api.messages.send, ...)
                │
                ▼
            VK API → пользователь получает ответ
```

Каждый вызов `messages.send` получает уникальный `random_id` — это защита от дублирования сообщений на стороне VK при повторных запросах.

Если текст длиннее 4096 символов (лимит VK), адаптер автоматически разбивает его на части по границам строк.

### Загрузка вложений

Входящие медиафайлы скачиваются и кешируются через стандартные утилиты Hermes:

| Тип VK          | Функция кеширования       | Передаётся агенту как |
|-----------------|---------------------------|----------------------|
| `photo`         | `cache_image_from_bytes`  | путь к файлу `.jpg`  |
| `audio_message` | `cache_audio_from_bytes`  | путь к `.ogg/.mp3`   |
| `doc`           | `cache_document_from_bytes` | путь к документу   |

Голосовые сообщения (`.ogg`) агент транскрибирует автоматически через встроенный инструмент распознавания речи Hermes.

Исходящие изображения загружаются через VK Upload API: сначала файл отправляется на сервер VK, затем отправляется как `attachment=photo{owner_id}_{photo_id}`.

## Требования

- Hermes Agent (установлен и настроен)
- Python 3.11+
- VK-сообщество с разрешением на работу с сообщениями
- Библиотека `vk_api`

```bash
pip install vk_api
```

## Установка

### 1. Получить токен VK-сообщества

1. Откройте своё сообщество ВКонтакте
2. **Управление → Работа с API → Создать ключ**
3. Включите разрешения: **Сообщения сообщества**, **Фотографии**, **Документы**
4. Включите **Long Poll API**: Управление → Сообщения → Бот-функции → Включить, затем Настройки Long Poll API → включить события `Входящее сообщение`

### 2. Скопировать плагин

```bash
# Linux / macOS / WSL
cp -r /путь/к/hermes-vk-connector/ ~/.hermes/plugins/vkontakte/

# Windows (PowerShell)
Copy-Item -Recurse .\hermes-vk-connector\ "$env:USERPROFILE\.hermes\plugins\vkontakte\"
```

Итоговая структура:

```
~/.hermes/plugins/vkontakte/
├── plugin.yaml
├── adapter.py
└── __init__.py
```

### 3. Задать переменные окружения

**Минимальная конфигурация:**

```bash
export VK_TOKEN="ваш_токен_сообщества"
```

**Полная конфигурация:**

```bash
# Обязательно
export VK_TOKEN="ваш_токен_сообщества"

# Белый список пользователей (пусто = разрешить всем)
export VK_ALLOWED_USERS="123456789,987654321"

# peer_id для крон-уведомлений (ваш user_id для ЛС или ID беседы)
export VK_HOME_CHANNEL="123456789"

# Версия VK API (по умолчанию 5.199)
export VK_API_VERSION="5.199"

# Разрешить всем пользователям (только для разработки!)
export VK_ALLOW_ALL_USERS="false"

# Таймаут Long Poll в секундах (по умолчанию 25)
export VK_POLLING_TIMEOUT="25"
```

### 4. Запустить Gateway

```bash
hermes gateway
```

Убедитесь, что VK появился в статусе:

```bash
hermes gateway status
```

## Использование

После запуска шлюза напишите боту личное сообщение в VK — агент ответит.

### Поддерживаемые типы сообщений

| Входящие (пользователь → бот) | Исходящие (бот → пользователь) |
|-------------------------------|-------------------------------|
| Текст                         | Текст (авторазбивка по 4096 символов) |
| Фотографии                    | Изображения                   |
| Голосовые сообщения           | Файлы-документы               |
| Документы и файлы             | Голосовые сообщения           |
| Беседы (групповые чаты)       | Индикатор «печатает…»         |

### Слэш-команды

Все стандартные команды Hermes работают через VK:

```
/new       — начать новый диалог (сбросить контекст)
/model     — сменить языковую модель
/retry     — повторить последний запрос
/skills    — список доступных навыков агента
/stop      — остановить текущую задачу
```

### Крон-уведомления

Если задан `VK_HOME_CHANNEL`, агент может отправлять плановые уведомления:

```
/schedule напомни мне о встрече завтра в 10:00
```

## Переменные окружения

| Переменная           | Обязательная | По умолчанию | Описание |
|----------------------|:---:|---|---|
| `VK_TOKEN`           | ✅  | —           | Токен API сообщества VK |
| `VK_ALLOWED_USERS`   | ❌  | (все)       | ID пользователей через запятую |
| `VK_HOME_CHANNEL`    | ❌  | —           | peer_id для крон-доставки |
| `VK_API_VERSION`     | ❌  | `5.199`     | Версия VK API |
| `VK_ALLOW_ALL_USERS` | ❌  | `false`     | Разрешить всех (только для тестов) |
| `VK_POLLING_TIMEOUT` | ❌  | `25`        | Таймаут Long Poll (секунды) |

## Ограничения

- **Нет inline-кнопок.** VK Bot API поддерживает клавиатуры, но текущая реализация не использует их — инструменты `clarify` и `approval` деградируют до текстового формата.
- **Нет редактирования сообщений.** VK не позволяет ботам редактировать отправленные сообщения — стриминг ответа реализован как последовательные сообщения.
- **Long Poll только для сообществ.** Личные страницы VK не поддерживают Bot Long Poll — нужно именно сообщество.
- **Синхронный VK API.** Библиотека `vk_api` не поддерживает async/await. Все вызовы блокирующие и выполняются в пуле потоков.

## Структура кода

```
adapter.py
├── check_vkontakte_requirements()    — проверка и lazy-установка vk_api
├── VKPlatformAdapter
│   ├── __init__()                    — парсинг конфига, env-переменных
│   ├── connect()                     — инициализация сессии, запуск потока
│   ├── disconnect()                  — остановка потока, очистка
│   ├── _poll_thread_fn()             — Long Poll в daemon-потоке
│   ├── _vk_event_to_message_event()  — маппинг VK-события → MessageEvent
│   ├── send()                        — отправка текста (с разбивкой)
│   ├── send_typing()                 — индикатор «печатает»
│   ├── send_image()                  — загрузка и отправка фото
│   ├── send_document()               — загрузка и отправка документа
│   ├── send_voice()                  — загрузка и отправка голосового
│   └── get_chat_info()               — имя и тип чата по peer_id
├── _split_text()                     — разбивка длинных сообщений
└── register(ctx)                     — точка входа плагин-системы
```
